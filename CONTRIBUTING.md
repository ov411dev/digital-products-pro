# Contributing

1. Create a branch from `main`.
2. Make focused changes.
3. Test on a current WordPress installation with PHP 8.0 or newer.
4. Update `CHANGELOG.md` when user-visible behavior changes.
5. Open a pull request with a concise explanation and screenshots when appropriate.

## Branch naming

- `feature/...`
- `fix/...`
- `docs/...`
- `release/...`
